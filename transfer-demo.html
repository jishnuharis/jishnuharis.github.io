FileTransferSystem

The relay connects two sockets and spawns a thread to pipe bytes in each direction — the server never buffers the file, just blindly forwards chunks as they arrive.

Design decisions
▸
Raw TCP sockets — no HTTP, no overhead, no framing
▸
Threading — one thread per direction for full-duplex relay
▸
IP anonymity — peers connect to the relay, never each other
▸
Zero buffering — chunks forwarded immediately as received
▸
Procfile deploy — hosted on Railway, always on
